Итоги проекта
Разработал API интернет-магазина на FastAPI, используя асинхронную работу с базой данных PostgreSQL через AsyncSession, SQLAlchemy и драйвер asyncpg. Проект охватывает ключевые функциональности для управления пользователями и товарами, с акцентом на безопасность и удобство тестирования.

Что сделал
Регистрация и аутентификация:
Реализовал эндпоинт POST /users/ для создания пользователей с ролями "seller" и "buyer".
Настроил POST /users/token для выдачи JWT-токенов: access_token (30 минут) и refresh_token (7 дней).
Использовал OAuth2PasswordBearer для аутентификации через логин и пароль.

Управление товарами:
Создал эндпоинты (например, POST /products/, GET /products/) для CRUD-операций с товарами.
Реализовал проверку роли "seller" через get_current_seller и привязку товаров к продавцу (seller_id).

Обновление токена:
Создал эндпоинт POST /refresh-token, чтобы обновлять refresh_token. А также, в виде домашнего задания, вы должны были реализовать эндпоинт, который принимает refresh_token и возвращает новый access_token.

Безопасность:
Проверяем is_active == True для пользователей при аутентификации и обновлении токена.
Используем SECRET_KEY для подписи и валидации JWT.
Рекомендация для продакшена: настроить HTTPS и CSRF-защиту.

Асинхронное программирование:
Работал с AsyncSession и await для запросов к базе (db.scalars, db.commit, db.refresh).
Избегал ошибок, таких как AttributeError из-за отсутствия await.
Аутентификация и авторизация:

Настроил OAuth2PasswordBearer для выдачи и проверки JWT-токенов.
Реализовал роли (role в JWT) и проверку владения (seller_id).

Тестирование API:
Освоил тестирование в Swagger UI.
